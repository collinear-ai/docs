## 🧪 What is Synthetic Data Generation?

**Synthetic Data Generation** allows you to create realistic, varied datasets from your source documents. Collinear AI processes your input files and generates synthetic examples that maintain the characteristics of your original data.

This helps you:

- Expand small datasets for better model training
- Create diverse test cases for evaluation
- Generate edge cases to test model robustness

# Introduction

Upload any supported file type and Collinear AI will generate synthetic variations while preserving the core characteristics of your original data.

---

## 🚀 Getting Started

To generate synthetic data:

1. Prepare your source file
2. Select a generation model
3. Specify replication count
4. Submit for processing

---

## 📄 Supported File Types

Collinear AI supports generation from:

- Text files (`.txt`)
- Log files (`.log`)
- CSS files (`.css`)
- JSON files (`.json`)
- Conversation transcripts
- Code files (`.py`, `.js`, etc.)

---

## ⚙️ Generation Parameters

### Model Selection
Choose from our specialized generation models:

1. **Base Generator** - Balanced variety and quality
2. **Structured Generator** - Better for code/logs
3. **Conversation Generator** - Optimized for chat data

### Replication Count
- **Range**: 1-100 variations per input
- **Recommendation**: Start with 5-10 for testing

### Quality Controls
- **Similarity Threshold**: Control how close to original
- **Diversity Boost**: Increase variation

---

## 📤 Output Format

Generated data is returned in standardized JSON format:

```json
{
  "original": "your original content",
  "synthetic": [
    "generated variation 1",
    "generated variation 2"
  ],
  "metadata": {
    "model": "generator-model-id",
    "parameters": {
      "temperature": 0.7,
      "diversity": 0.8
    }
  }
}
